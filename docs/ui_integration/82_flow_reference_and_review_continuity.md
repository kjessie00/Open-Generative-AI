## 한 줄 결론

이미지 결과를 먼저 검토하고 필요한 항목만 다시 만드는 작업대 흐름은 실제 Electron에서 저장·재실행까지 통과했다. Flow의 마지막 장면 참조 0장과 중간 장면 참조 2장 경로도 실제 작업창에서 생성 버튼을 누르기 직전까지 모두 통과했다.

## 이번 변경

- 이미지 결과가 연결돼도 `사용` 또는 `다시 만들기`를 고르지 않았다면 영상 작업보다 결과 검토를 먼저 안내한다.
- 생성 준비 화면의 접힌 자료 구역을 열지 않아도 `결과 검토로` 버튼이 보인다.
- Flow 영상 참조는 장면 순서대로 구성한다.
  - 마지막 전 장면까지: 현재 장면 이미지 + 다음 장면 이미지, 정확히 2장
  - 마지막 장면: 참조 0장
  - Grok과 Replicate: 기존처럼 현재 장면 이미지 1장
- 일부 장면만 다시 만들 때도 재제작 목록의 다음 항목이 아니라 전체 스토리보드의 실제 다음 장면을 참조한다.
- Flow 실행 미리보기는 Electron main에서만 절대 경로와 CDP 연결을 조합한다. renderer에는 경로, CDP 주소, 내부 차단 코드가 노출되지 않는다.
- Flow 명령은 `shell=false`, `preview_only=true`, `live_submit_allowed=false`, `--no-submit`으로 고정한다.

# 이미지·영상 프롬프트 에이전트 협업

## 한 줄 결론

이미지와 영상 카드에서 프롬프트를 직접 고치거나 로컬 Codex에게 선택한 항목 하나만 수정해 달라고 요청할 수 있다. 현재 내용과 수정안을 같은 카드에서 비교해 적용하거나 그대로 유지하며, 어느 결정도 이미지·영상 생성을 시작하지 않는다.

## 사용자 흐름

1. `생성 준비 → 이미지 작업` 또는 `영상 작업`을 연다.
2. 필요한 카드의 `프롬프트 수정`을 연다.
3. `현재 프롬프트`를 직접 수정하거나 `어떻게 바꿀까요?`에 요청을 적고 `수정안 만들기`를 누른다.
4. 에이전트는 전체 카드 순서와 프롬프트를 문맥으로 읽지만 선택한 카드의 프롬프트 하나만 제안한다.
5. 같은 카드의 `현재 프롬프트`와 `에이전트 수정안`을 비교한다.
6. `수정안 적용` 또는 `현재 내용 유지`를 고른다.

결과가 이미 연결된 항목은 먼저 `다시 만들기`를 선택해야 수정 요청을 보낼 수 있다. 영상 생성 도구는 사용자가 직접 고르며 에이전트 수정안이 바꾸지 않는다.

## 화면 원칙

- 새 모달·탭·상태 배지를 추가하지 않았다.
- 짧은 한국어 문구만 사용하고 요청 ID, 파일 경로, 해시, 모델명, 영문 오류를 표시하지 않는다.
- 수정안이 있는 카드만 화면 한 줄 폭으로 넓어지고 다른 카드는 기존 순서를 유지한다.
- 작은 화면에서는 비교 영역과 결정 버튼이 세로로 배치된다.
- native `details`, `textarea`, `button`, `checkbox`를 사용한다.
- 수정안은 `readonly`라 읽고 복사할 수 있으며, 진행 상태는 `aria-live`로 전달한다.

## 실행·저장 계약

- renderer는 고정된 `window.filmPipeline` IPC만 호출하고 셸이나 파일 경로를 소유하지 않는다.
- 요청 시 현재 프롬프트를 먼저 저장하고 계획 revision과 대상 카드 상태를 고정한다.
- private sidecar의 요청·전체 계획 snapshot·수정안·결정 receipt는 `0700` 디렉터리와 `0600` 파일로 저장된다.
- 로컬 Codex는 기존 고정 no-tool, read-only, ephemeral, `gpt-5.3-codex-spark` `xhigh` 계약으로 텍스트만 수정한다.
- 모델 출력은 `proposed_prompt`와 짧은 `summary`만 허용한다. task identity, 결과 상태, 참조, 영상 생성 도구는 모델이 소유하지 않는다.
- JSON 묶음, `request_instruction/target/plan_context` 같은 내부 필드, 코드 블록, 원문과 같은 수정안은 거부한다.
- 적용 시 현재 revision과 대상 문맥을 다시 확인하고 기존 plan save 경로로 대상 `prompt` 하나만 교체한다.
- 보류는 계획을 쓰지 않고 receipt만 남긴다.
- 요청·비교·적용·보류 전 과정의 `generation_executed`는 `false`다.

## 실제 검증

격리한 실제 프로젝트에서 로컬 Codex를 직접 실행했다.

- 이미지 프롬프트 실제 수정 1회: 모델 호출됨, 대상 프롬프트 변경, 다른 항목 불변
- 영상 프롬프트 실제 수정 1회: 모델 호출됨, 대상 프롬프트 변경, Flow 선택 유지
- 실제 이미지 생성 provider 호출 0회
- 실제 영상 생성 provider 호출 0회
- 로컬 fixture 이미지는 영상 계획 연결 조건만 검증하기 위해 사용했고 실제 생성 결과로 주장하지 않는다.
- 최종 영상 프롬프트는 일반 한국어 본문이며 JSON 묶음과 내부 필드가 없다.

Computer Use로 실제 macOS Electron 창을 열어 다음을 확인했다.

- 이미지와 영상 카드에 직접 편집과 `에이전트에게 수정 요청`이 함께 보임
- 결과 연결 항목은 `다시 만들기` 전 요청이 막힘
- 현재 내용과 수정안이 나란히 보이고 내부 식별 정보나 상태 배지가 없음
- 실제 IPC의 `현재 내용 유지`가 계획을 바꾸지 않고 완료됨
- 실제 IPC의 `수정안 적용`이 선택 프롬프트를 저장하고 `생성은 시작하지 않았습니다`로 완료됨

Computer Use 검증 중 두 결함을 찾아 수정했다.

1. 영상 모델이 내부 요청 JSON을 프롬프트 문자열로 되돌리는 경우를 거부하도록 출력 검증을 강화했다.
2. renderer client 공개 객체에서 새 prompt-agent 함수가 빠져 결정 버튼이 실패하던 문제를 고치고 회귀 테스트를 추가했다.

비교 UI 화면은 실제 Electron 위에서 `MOCK suggestion`을 사용했다. 에이전트 실제 호출과 프롬프트 적용 검증은 별도 실제 로컬 Codex 실행으로 확인했다.

| 단계 | 앱 안에서 가능한 일 | 아직 별도인 일 |

| 기획·대본 | 직접 수정, 실제 로컬 에이전트 요청, 비교, 적용·유지 | 없음 |
| 인물·장소·장면 설계 | 직접 수정, 실제 로컬 에이전트 요청, 비교, 적용·유지 | 없음 |

| 이미지 결과 | 순서대로 확인, 로컬 DST 결과 연결, 필요한 항목만 다시 만들기 | 결과 품질의 사람 승인 |
| 영상 프롬프트 | 카드별 직접 수정, 생성 도구 직접 선택, 실제 로컬 에이전트 요청, 비교, 적용·유지 | Flow/Grok/Replicate/ByteDance live 생성 제출 |
| 영상 결과 | 로컬 완료 영상 연결, 필요한 항목만 다시 만들기, 클립 QA와 사용 구간 선택 | 결과 품질의 사람 승인과 외부 업로드 |
| 마무리 | 선택 구간 기반 로컬 최종 편집·보고 | 최종 납품 승인 |

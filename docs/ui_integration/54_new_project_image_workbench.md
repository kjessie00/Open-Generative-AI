# 새 프로젝트 이미지 작업대

## 결론

`3 생성 준비 → 이미지 작업`은 저장된 설계에서 이미지 작업을 `인물 → 장소 → 장면` 순서로 자동 구성한다. 사용자는 각 프롬프트를 직접 고치고, DST 결과를 카드에 연결하고, 전체 결과를 한눈에 본 뒤 필요한 카드만 `다시 만들기`로 고를 수 있다.

`DST 작업 준비`는 현재 안전한 로컬 준비 단계다. 순서와 프롬프트를 앱 전용 저장소에 기록하지만 DeepSearchTeam, 브라우저, 모델, API 또는 이미지 생성을 실행하지 않는다.

## 사용자 흐름

1. `1 기획·대본`과 `2 설계`를 저장한다.
2. `3 생성 준비 → 이미지 작업`에서 인물 시트, 장소 시트, 장면 이미지를 순서대로 확인한다.
3. 필요하면 카드의 `프롬프트 수정`을 열어 직접 고치고 `프롬프트 저장`을 누른다.
4. `DST 작업 준비`로 아직 결과가 없거나 다시 만들기로 고른 항목만 순서대로 저장한다.
5. DST가 만든 완료 묶음이 로컬 결과 폴더에 있으면 `DST 결과 연결`에서 시간과 이미지 수를 보고 미리보기한다.
6. `이 결과 연결`로 카드를 채우고, 결과가 마음에 들지 않는 카드만 `다시 만들기`로 고른다.
7. 다시 `DST 작업 준비`를 누르면 선택한 카드와 아직 비어 있는 카드만 다음 작업 목록에 들어간다.

상단에는 `완료 / 다시 만들기 / 다음 작업`만 한 줄로 보인다. 내부 토큰, 해시, 경로, provider 이름, 영문 상태 코드는 표시하지 않는다. 기존 제작 폴더가 없을 때는 MOCK 자료 대신 `연결된 기존 제작 폴더가 없습니다`를 표시한다.

## 작업 계약

- 작업 순서: 모든 인물 시트 → 모든 장소 시트 → 모든 장면 이미지
- 장면 참조: 해당 장면의 인물 시트와 장소 시트 작업을 앞선 참조로 고정
- 기본 프롬프트: 9:16, 인물·의상·장소 연속성, 텍스트·로고·워터마크 제외
- 직접 수정 가능 값: 프롬프트
- 상태: `이미지 필요`, `결과 연결됨`, `다시 만들기`
- 설계가 바뀌면 새 설계에서 작업을 다시 도출하고, 사용자가 저장하기 전에는 준비·결과 연결·재제작 변경을 차단
- 연결 결과는 앱 전용 `userData`로 복사하며 production 폴더를 수정하지 않음
- 디렉터리 권한 `0700`, 파일 권한 `0600`; renderer 상태와 IPC는 경로를 노출하지 않음

## 실제 Electron 검증

격리된 실제 Electron `userData`와 유효한 합성 DST 완료 묶음을 사용해 다음 전 과정을 실행했다.

1. 인물 2명, 장소 2곳, 장면 4개의 저장된 설계에서 정확히 8개 작업 도출
2. 작업 종류가 `인물 2 → 장소 2 → 장면 4` 순서인지 확인
3. 첫 프롬프트를 실제 화면에서 수정·저장
4. 첫 `DST 작업 준비`가 8개를 기록하고 생성·모델 호출은 하지 않았는지 확인
5. 실제 renderer → preload → main 경로로 8개 카드에 로컬 결과를 연결하고 이미지 디코딩 확인
6. 마지막 장면 한 개만 `다시 만들기`로 선택
7. 두 번째 준비 목록이 정확히 한 개만 포함하는지 확인
8. Electron 완전 종료·재실행 후 결과 8개, 재제작 1개, 디코딩 이미지 8개를 복원

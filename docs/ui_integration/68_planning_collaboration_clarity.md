# 기획·스크립트 협업 화면 정리

## 한 줄 결론

기획과 스크립트마다 `직접 수정`과 `에이전트에게 요청`을 같은 화면에서 바로 고를 수 있다. 저장·에이전트 실행·수정안 비교 계약은 그대로 두고, 반복 안내와 빌드 명령만 기본 화면에서 치웠다.

## 사용 흐름

1. `1 기획·대본`을 연다.
2. 직접 고치려면 `직접 수정` 아래 본문을 편집하고 `직접 저장`을 누른다.
3. 도움을 받으려면 `에이전트에게 요청` 아래 `어떻게 바꿀까요?`에 원하는 변경을 적고 `에이전트 작업 시작`을 누른다.
4. 수정안이 도착하면 현재 내용과 비교해 `수정안 적용` 또는 `현재 내용 유지`를 선택한다.

에이전트 요청 전에 현재 전체 초안이 먼저 저장되고, Electron main이 요청 시점 revision과 private snapshot을 소유한다. renderer는 command, cwd, 모델, 경로, API key를 받지 않으며 에이전트 결과도 자동 적용하지 않는다.

## 화면 단순화

- 기획·스크립트 각각에 `직접 수정`과 `에이전트에게 요청` 제목을 명시했다.
- 단계마다 반복되던 기본 대기 문구를 숨겼다. 입력 누락, 요청 대기, 실패처럼 행동이 필요한 상태만 다시 표시된다.
- “제작·생성을 시작하지 않는다”는 안내를 화면 상단 한 번으로 줄였다.
- `빌드 명령 복사`와 명령 본문은 접힌 `고급: 빌드 명령` 안으로 옮겼다.
- 상태 배지는 추가하지 않았다.

## 실제 검증

격리된 macOS Electron 앱에서 다음을 확인했다.

- `직접 수정` 2개, `직접 저장` 2개
- `에이전트에게 요청` 2개, `에이전트 작업 시작` 2개
- 기획·스크립트 직접 입력 후 `직접 저장됨` 확인
- 앱 완전 종료·재실행 뒤 두 본문 exact 복원
- 에이전트 요청 입력칸에 자연어 요청 직접 작성 가능
- 빌드 명령은 기본 상태에서 접혀 있음
- 반복 대기 문구 0, 생성 안 함 안내 1회
- 에이전트 모델 호출 0, 이미지·영상 생성 0

# 새 프로젝트 영상 작업대

## 결론

`3 생성 준비 → 영상 작업`에서 설계된 장면을 순서대로 보고, Flow·Grok·Replicate·ByteDance 중 생성 도구를 고르고, 프롬프트를 직접 고치고, 완료된 로컬 영상을 연결할 수 있다. 연결한 영상은 카드에서 바로 재생하며, 사용자는 필요한 장면만 `다시 만들기`로 고를 수 있다.

`영상 작업 준비`는 순서·도구·프롬프트만 앱 전용 저장소에 기록한다. 유료 생성, Flow 제출, Grok 호출, API, 브라우저 자동화, 업로드는 실행하지 않는다.

## 한눈에 보는 사용 흐름

1. `1 기획·대본`에서 직접 작성하거나 에이전트에게 수정을 요청한다.
2. `2 설계`에서 인물·장소·장면을 저장한다.
3. `이미지 작업`에서 인물 시트 → 장소 시트 → 장면 첫 프레임을 준비하고 연결한다.
4. `영상 작업`에서 장면별 도구와 프롬프트를 확인한다.
5. 완료 영상을 미리보고 카드에 연결한다.
6. 전체를 한눈에 보고 필요한 카드만 `다시 만들기`로 선택한다.
7. `영상 작업 준비`를 다시 누르면 선택한 항목과 빈 항목만 다음 작업 목록에 들어간다.

상단은 `완료 / 다시 만들기 / 다음 장면`만 표시한다. 카드에서는 장면 이름, 생성 도구, 프롬프트, 영상, 재제작 선택만 보이며 경로·해시·토큰·영문 내부 상태는 표시하지 않는다.

## 현재 실제로 가능한 파이프라인

| 단계 | 현재 작업대에서 가능한 일 | 현재 경계 |
|---|---|---|
| 기획·대본 | 직접 수정·저장, 에이전트 요청 저장, 수정안 비교·적용 | 에이전트 모델 자동 실행은 별도 연결 필요 |
| 설계 | 인물·장소·장면 직접 수정·저장, 에이전트 요청·수정안 적용 | 외부 모델 실행은 별도 연결 필요 |
| 이미지 | 인물 → 장소 → 장면 순서, 프롬프트 수정, DST 완료 결과 미리보기·연결·재제작 선택 | DST 실제 생성 제출·진행 감시는 미연결 |
| 영상 | Flow·Grok·Replicate·ByteDance 선택, 프롬프트 수정, 로컬 완료 결과 미리보기·연결·재생·재제작 선택 | 실제 생성 제출·진행 감시는 미연결 |
| 클립 선택·마무리 | 기존 production의 QA, 채택 구간, 로컬 최종 편집·보고 | 실제 산출물 품질과 Jessie 최종 승인은 별도 |

## 저장과 복원 계약

- 영상 계획은 장면 순서와 첫 프레임 이미지 결과에 고정된다.
- 설계나 이미지 계획이 바뀌면 기존 영상 계획을 그대로 승계하지 않고 명시적 재저장을 요구한다.
- 완료 영상은 renderer가 파일을 직접 읽지 않고 Electron main이 재검증·해시·ffprobe 후 앱 전용 영역으로 스트림 복사한다.
- 최대 512 MiB 원본을 연결하며, 32 MiB renderer 미리보기 한계를 원본 연결 한계로 사용하지 않는다.
- 프로세스마다 바뀌는 후보 토큰은 저장하지 않고, provider·원본 해시·길이·크기·출처 종류만 private manifest에 보존한다.
- 앱 전용 디렉터리는 `0700`, 계획·대기열·결과 파일은 `0600`이다.

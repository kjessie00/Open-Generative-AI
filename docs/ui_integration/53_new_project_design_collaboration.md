# 새 프로젝트 설계 보드와 에이전트 협업

## 결론

`2 설계`의 스토리보드 화면을 새 프로젝트용 설계 작업대로 바꿨다. 사용자는 `인물 시트 → 장소 시트 → 장면 카드` 순서로 전체 설계를 한눈에 보고 직접 수정·저장하거나, 같은 화면에서 에이전트에게 전체 보드 수정을 요청할 수 있다. 기존 이미지·영상 결과 검토는 삭제하지 않고 `생성 결과 검토` 안에 접어 분리했다.

## 사용자 흐름

1. `1 기획·대본`에서 기획과 스크립트를 직접 저장한다.
2. `2 설계`에서 인물, 장소, 장면을 순서대로 입력한다.
3. 필요한 내용을 직접 고친 뒤 `직접 저장`한다.
4. 전체 구조를 다듬고 싶으면 `에이전트에게 요청`에 짧게 적는다.
5. 수정안이 도착하면 `현재 설계`와 `에이전트 수정안`을 나란히 본다.
6. `수정안 적용`은 전체 설계 보드를 바꾸고, `보류`는 현재 설계를 유지한다.
7. 적용·보류 결과와 직접 저장한 설계는 앱을 완전히 종료해도 복원된다.

화면에는 인물·장소·장면 바로가기, 직접 저장, 에이전트 요청, 수정안 확인·적용·보류만 보인다. 요청 ID, 토큰, 해시, 내부 상태, 모델/provider, 파일 경로는 표시하지 않는다. 배지 대신 `요청 저장됨 · 아직 실행 전`, `수정안이 도착했습니다`, `보류함 · 현재 설계는 그대로` 같은 짧은 한국어 문장을 사용한다.

## 설계 보드 계약

- 인물 시트: 이름, 역할, 외형, 의상, 연속성
- 장소 시트: 이름, 공간, 조명, 소품, 연속성
- 장면 카드: 제목, 핵심 장면, 등장인물, 장소, 길이, 첫 화면, 행동, 카메라, 조명, 소리
- 최대 수: 인물 12, 장소 12, 장면 20
- 최소 저장 조건: 인물·장소 이름과 ID, 장면 제목·핵심 장면·장소·길이·행동
- 처음 비어 있는 보드에서도 저장을 강제하지 않고 바로 첫 에이전트 요청을 남길 수 있다.
- 직접 편집 중이면 오래된 수정안의 적용을 막고, 현재 전체 보드를 먼저 저장한 뒤 새 요청을 만든다.

## 실행과 보안 경계

- renderer는 파일이나 셸을 직접 다루지 않고 `window.filmPipeline`만 호출한다.
- Electron main은 앱 전용 `userData` 아래에 보드, 요청 스냅샷, 수정안, 결정 영수증을 저장한다.
- 디렉터리는 `0700`, 파일은 `0600`이며 queue의 예상 밖 파일·심볼릭 링크·느슨한 권한은 다음 요청 전에 차단한다.
- 요청 시점의 기획·대본·설계 전체를 먼저 불변 스냅샷으로 만들고 요청 파일을 마지막에 발행한다.
- 외부 로컬 에이전트는 `scripts/design_agent_handoff.mjs prepare`로 정확한 패킷을 읽고 `publish`로 완성된 전체 보드를 한 번만 발행한다.
- 앱 내부 요청은 계속 `executed:false`, `model_called:false`다. 앱은 모델, API, API 키, 브라우저 자동화, 생성, 업로드, production 쓰기를 실행하지 않는다.
- 수정안 적용 직전에 기획 revision과 전체 설계 revision을 다시 비교한다. 직접 저장 또는 기획 변경이 생기면 오래된 수정안을 적용하지 않는다.
- 적용·보류는 불변 영수증으로 남고, 보드 저장 뒤 영수증 발행이 끊겨도 안전하게 복구한다.

## 실제 Electron 검증

격리된 실제 Electron `userData`에서 다음 전 과정을 실행했다.

1. `문의 32통, 사장 몫 0원` 기획·대본 저장
2. 포장이사 사장, 비 오는 고층 아파트, 첫 장면을 직접 입력·저장
3. 현재 Codex가 실제 요청 패킷의 기획·대본·1장면 보드를 읽음
4. 인물 2명, 장소 2곳, 장면 4개로 갈등·숨은 비용·체크리스트 전환·64만원 결과를 연결한 전체 수정안 발행
5. 실제 UI 비교 후 수정안 적용
6. 적용된 4장면 보드를 다시 패킷으로 읽고 첫 장면의 미끄럼·낙하 위험과 마지막 `32통·0원 → 11통·64만원` 대비를 강화한 두 번째 수정안 발행
7. 실제 UI에서 두 번째 수정안 보류
8. Electron 완전 종료·재실행 후 첫 수정안의 4장면 보드 exact 복원, 최신 수정안 `held`, 적용 수 1 확인

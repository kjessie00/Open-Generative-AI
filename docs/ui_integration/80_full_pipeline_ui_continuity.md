## 수정한 연속성 문제

- 샘플 fallback을 제거하고 production을 선택하지 않은 새 프로젝트는 실제 저장 상태만 표시한다.
- 기존 production을 선택한 경우에는 legacy production 상태와 기획 화면만 표시하고 새 프로젝트 초안·에이전트 폼을 숨겨 두 흐름이 서로 섞이지 않는다.
- `샷 설계`와 `모션 보드`를 현재 설계의 직접 편집·저장·에이전트 요청 화면으로 연결했다.
- `작업 진행`이 빈 receipt일 때 현재 이미지·영상 plan/result/review 상태를 투영한다.
- `프롬프트 팩`과 `검토 게이트`가 현재 새 프로젝트 결과·결정을 표시한다.
- 이미지 완료 후 영상, 영상 완료 후 클립 선택으로 가는 다음 버튼을 추가했다.
- 영상 후보를 빠르게 바꿀 때 늦은 미리보기는 폐기하고, 연결 요청은 한 번만 실행한다.
- 초기 복원·저장·새로고침이 역순으로 끝나도 draft/design/image/video/clip/final 상태마다 최신 응답만 반영한다.
- 클립 저장 중 직접 수정이나 최종 편집 새로고침이 겹쳐도 최신 클립과 최신 최종 편집 상태를 각각 보존한다.
- 설정의 내부 경로와 기술 배지를 기본 화면에서 숨겼다.

## 남은 제품 경계

- DST: 로컬 완료 묶음 비교·정확한 이미지 연결은 PASS. `dst image`에 no-submit 모드가 없어 라이브 생성은 미연결.
- Flow: no-submit 미리보기와 `0장 또는 2장` 참조 검사는 있으나 비공개 runtime/result staging과 제출은 미연결.
- Grok: 미리보기·완료 결과 가져오기는 있으나 길이·화면비·참조·non-submit 계약을 만족하는 라이브 실행은 미연결.
- Replicate: 요청 미리보기·결과 영수증·로컬 결과 연결은 PASS. 인증 제출·상태 조회·다운로드 실행기는 미연결.
- ByteDance: 결과 receipt 계약만 있고 직접 생성 adapter와 실제 결과 증거는 없음.

따라서 **로컬 작업대 UI 파이프라인과 최종 로컬 검토는 PASS**, **실제 provider 생성 전체 파이프라인은 아직 PARTIAL**이다. 생성물의 창작 품질과 Jessie의 최종 승인은 이 기술 검증과 별도다.

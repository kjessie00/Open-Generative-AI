# 새 프로젝트 결과 사용·재제작 결정

## 목적

새 프로젝트의 인물·장소·장면 이미지와 장면 영상을 한 작업대에서 보고, 각 결과를 `이 결과 사용` 또는 `다시 만들기`로 명시적으로 결정한다. 결과 파일이 연결된 사실과 사람이 품질을 승인한 사실을 분리한다.

## 구현 계약

- 연결된 결과의 기본 상태는 `확인 필요`이다.
- `이 결과 사용`은 별도 비공개 `review-decisions.json`에 현재 작업·결과·설계 revision을 묶어 저장한다.
- `다시 만들기`는 기존 재제작 선택에 저장하며, 같은 결과의 사용 결정은 제거한다.
- 인물·장소 기준 이미지를 사용하기로 확인해야 장면 이미지를 준비할 수 있다.
- 모든 장면 이미지를 사용하기로 확인해야 영상 작업을 준비할 수 있다.
- 이미지 재제작으로 상위 revision이 바뀌면 영상 계획을 숨기고, 이미지를 다시 사용하면 동일한 결정적 revision의 영상 결과·결정을 복원한다.
- renderer에는 경로·토큰·해시를 노출하지 않고, pathless Electron IPC만 사용한다.

## UI

- 상태는 `확인 필요`, `사용하기로 확인함`, `다시 만들기로 선택됨`의 짧은 한글만 표시한다.
- 필터는 `전체`, `검토할 결과`, `다시 만들기`이다.
- 각 결과 카드에는 `이 결과 사용`, `다시 만들기`, `작업 열기`만 둔다.
- 배지는 추가하지 않았고, 상단에서 `확인 필요 N · 사용 N · 다시 만들기 N`을 한 줄로 집계한다.
- 다음 행동은 이미지 검토를 영상보다 우선한다.

## 실제 검증

격리된 Electron userData와 로컬 synthetic PNG/MP4 fixture를 사용했다.

- 시작 상태: 이미지 3개 연결, 영상 1개 연결, 사용 2개, 확인 필요 이미지 1개.
- 장면 이미지를 사용하면 영상 카드가 나타나고 `영상 1개 확인`으로 이동했다.
- 영상과 장면 이미지를 각각 다시 만들기로 바꿀 때 형제 승인 보존과 상위 의존성 차단을 확인했다.
- 두 결과를 다시 사용하면 `확인 필요 0 · 사용 4 · 다시 만들기 0`으로 복귀했다.
- `검토할 결과` 필터는 빈 결과만 표시했고, 키보드 `Tab`과 `Return`으로 필터를 조작했다.
- 앱을 완전히 종료하고 재실행한 뒤 이미지 3개와 영상 1개의 사용 결정이 모두 복원됐다.
- 두 review 파일의 권한은 `0600`이었다.
- 검증기 결과: 외부 호출 0, 모델 호출 0, 생성 실행 0, provider 생성 호출 0.
